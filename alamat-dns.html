6. apa yang dimaksud dengan dns dan apa perbedaan public dns dengan dns default


DNS yaitu Sebuah sistem yang menyimpan informasi tentang nama host ataupun nama domain dalam bentuk basis data tersebar (distributed database) di dalam jaringan komputer.

Jawaban:

DNS atau Domain Name System berfungsi sebagai penerjemah/pengubah dari alamat situs ke bentuk IP address.

Simplenya DNS adalah buku telepon agar perangkat kita dapat menemukan IP address dari suatu situs web dengan lebih cepat.

Contoh DNS

Beberapa DNS publik yang sudah sangat umum yaitu DNS Google (8.8.8.8) dan OpenDNS (208.67.222.222). Banyak penyedia Public DNS lainnya yang memberikan layanan DNS secara gratis seperti Cloudflare, AdGuard, Quad9, Verisign, dll. yang menyediakan keunggulannya masing-masing.

Semua tergantung User dalam memilih ingin menggunakan DNS yang mana.

Perbedaan

DNS Publik atau Public DNS memiliki beberapa keunggulan tersendiri tergantung dari penyedia DNS-nya. Terkadang menggunakan Public DNS jauh lebih menguntungkan dalam beberapa hal dibandingkan menggunakan DNS default bawaan provider.

Contoh Penggunaan

Menggunakan Public DNS seperti Google DNS lebih menguntungkan daripada menggunakan DNS bawaan provider karena server Google sendiri sudah sangat canggih, jauh lebih maju dibanding server yang digunakan provider internet tertentu sehingga dalam beberapa kasus jika kita menggunakan Google DNS atau DNS publik lainnya maka aktivitas browsing kita bisa jauh lebih lancar dan cepat.

Lalu biasanya DNS bawaan dari provider internet kita sudah dimodifikasi agar menampilkan iklan yang mempromosikan produk & layanan mereka. Beberapa pengguna pastinya merasa terganggu dengan iklan-iklan ini. Hal ini tentunya tidak ada jika menggunakan Public DNS dari Google, OpenDNS, Cloudflare, dsb.

Catatan

Yang perlu diperhatikan adalah pastikan untuk selalu menggunakan Public DNS dari penyedia yang memang terpercaya seperti Google, Cloudflare, OpenDNS, dan sejenisnya. Hal ini karena aktivitas browsing dapat dicatat oleh server DNS. Agar tidak disalah-gunakan maka harus menggunakan penyedia Public DNS yang terpercaya.

19. Hal atau faktor apa yang menjadi pemisah dari 2 bagian alamat IP yaitu network dan host ? Subnet Mask Broadcast Address Default Gateway DNS Server​


Jawaban:

Hal atau faktor yang menjadi pemisah dari dua bagian alamat IP, yaitu network dan host, adalah:

1. Subnet Mask: Subnet mask digunakan untuk memisahkan bagian network dan host dalam alamat IP. Subnet mask adalah sebuah bilangan biner atau desimal yang digunakan untuk menentukan sejauh mana bagian alamat IP yang digunakan untuk mengidentifikasi jaringan (network) dan bagian yang digunakan untuk mengidentifikasi perangkat (host). Subnet mask memberikan petunjuk tentang bagian mana dari alamat IP yang merupakan bagian network dan bagian host.

2. Broadcast Address: Broadcast address adalah alamat IP khusus yang digunakan untuk mengirim pesan broadcast ke semua perangkat dalam suatu jaringan. Dalam jaringan yang menggunakan subnetting, broadcast address digunakan untuk mengirim pesan broadcast dalam subnet tertentu. Broadcast address adalah alamat IP dengan bagian host yang seluruh bitnya diatur ke nilai biner 1.

3. Default Gateway: Default gateway adalah alamat IP perangkat jaringan yang digunakan sebagai pintu gerbang (gateway) untuk menghubungkan satu jaringan dengan jaringan lainnya. Default gateway biasanya merupakan alamat IP perangkat router yang menghubungkan jaringan lokal dengan jaringan eksternal atau internet. Default gateway memungkinkan perangkat dalam jaringan untuk mengirimkan paket data ke tujuan di luar jaringan lokal.

4. DNS Server: DNS server (Domain Name System server) adalah server yang bertanggung jawab untuk menerjemahkan nama domain menjadi alamat IP yang sesuai. DNS server membantu dalam proses resolusi nama domain menjadi alamat IP yang diperlukan untuk mengarahkan koneksi ke tujuan yang diinginkan.

Semua faktor di atas berperan dalam memisahkan dan mengarahkan bagian network dan host dalam alamat IP serta dalam pengiriman data antara jaringan dan ke tujuan yang diinginkan.

Alamat IP (Internet Protocol Address atau sering disingkat IP) adalah deretan angka biner antar 32-bit sampai 128-bit yang dipakai sebagai alamat identifikasi untuk tiap komputer host dalam jaringan Internet. Panjang dari angka ini adalah 32-bit (untuk IPv4 atau IP versi 4), dan 128-bit (untuk IPv6 atau IP versi 6) yang menunjukkan alamat dari komputer tersebut pada jaringan Internet berbasis TCP/IP.

Pengertian Domain Name System (DNS)

Domain Name System (DNS) adalah distribute database system yang digunakan untuk pencarian nama komputer (name resolution) di jaringan yang mengunakan TCP/IP (Transmission Control Protocol/Internet Protocol). DNS biasa digunakan pada aplikasi yang terhubung ke Internet seperti web browser atau e-mail, dimana DNS membantu memetakan host name sebuah komputer ke IP address.

Penjelasan:

Hubungnnya adalah DNS dan IP adress adalah sama sama sebagai penamaan sistem, bedana jika DNS terdapat nama contohnya (namadomain.com) sedangkan ip dari namadomain.com (192.178.0.1)

40. Tuliskan Langkah-langkah menambahkan alamat DNS di Windows agar client bisa mengakses Domain yang sudah dikonfigurasi.​


Berikut adalah langkah-langkah menambahkan alamat DNS di Windows agar client bisa mengakses domain yang sudah dikonfigurasi:

Buka Control Panel pada Windows dan pilih Network and Sharing Center.

Klik pada opsi Change adapter settings.

Klik kanan pada adapter yang sedang digunakan dan pilih Properties.

Pilih Internet Protocol Version 4 (TCP/IPv4) dan klik Properties.

Pilih opsi Use the following DNS server addresses.

Masukkan alamat DNS yang ingin ditambahkan pada Preferred DNS server dan Alternate DNS server.

Klik OK untuk menyimpan perubahan.

Ulangi langkah-langkah ini pada setiap komputer client yang ingin mengakses domain yang sudah dikonfigurasi.

Setelah menambahkan alamat DNS pada komputer client, pengguna dapat mengakses domain yang sudah dikonfigurasi melalui web browser atau aplikasi lainnya yang memerlukan koneksi internet. Pastikan untuk memeriksa kembali setiap konfigurasi DNS pada server dan client untuk memastikan bahwa semuanya sudah benar.
