Jawaban: Cron daemon atau cron service.

• • • • • • • • • • • • • • • • • • • • • • • • • • • • • •

PEMBAHASAN

Cron daemon atau cron service adalah program yang berjalan pada sistem operasi Unix atau Linux yang bertanggung jawab untuk menjalankan pekerjaan yang dijadwalkan dalam file crontab. Setiap pengguna dapat membuat file crontab untuk menjadwalkan tugas-tugas yang harus dilakukan secara otomatis pada waktu tertentu. Cron daemon memeriksa file crontab untuk setiap pengguna secara teratur dan menjalankan tugas-tugas yang dijadwalkan pada waktu yang ditentukan dalam file crontab. Dengan menggunakan cron daemon, pengguna dapat memastikan bahwa tugas-tugas yang dijadwalkan akan dijalankan pada waktu yang diinginkan dengan akurasi dan keandalan yang tinggi.

13. 2. Tuliskan Komponenn Sistem Operasi a. File b. Kernel c. User Interface


Jawaban:

a. File

File disini merupakan file-file yang dibentuk atau dijalankan oleh sistem operasi. File disini juga berarti sistem berkas dan file system yang dimiliki oleh suatu sistem operasi.

b.Kernel

Kernel merupakan suatu software (kumpulan program) yang membentuk sistem dan memiliki tugas melayani bermacam program aplikasi untuk mengakses hardware komputer secara aman dan terkendali. Karena akses terhadap hardware terbatas, sedangkan ada lebih dari satu program yang harus dilayani dalam waktu yang bersamaan, maka kernel juga bertugas untuk mengatur agar kapan dan berapa lama suatu program dapat menggunakan satu bagian hardware tersebut. Hal tersebut dinamakan sebagai ”multiplexing”.

Ada 4 kategori kernel:

· Monolitich kernel. Kernel yang menyediakan abstraksi hardware yang kaya dan powerful.

· Microkernel. Kernel yang menyediakan hanya sekumpulan kecil abstraksi hardware sederhana, dan menggunakan aplikasi-aplikasi yang disebut sebagai server untuk menyediakan fungsi-fungsi lainnya.

· Hybrid (modifikasi dari microkernel). Kernel yang mirip microkernel, tetapi ia juga memasukkan beberapa kode tambahan di kernel agar ia menjadi lebih cepat

· Exokernel. Kernel yang tidak menyediakan sama sekali abstraksi hardware, tapi ia menyediakan sekumpulan library yang menyediakan fungsi-fungsi akses ke hardware secara langsung atau hampir-hampir langsung.

c. User Interface (Shell)

Sebuah sistem operasi memiliki karakteristik (bentuk) interface (tampilan) yang menjadi interaksi antar user dengan komputer. Bentuk umum user interface yang ada, adalah:

1. Command Line Interface atau command line interpreter (CLI)

2. Graphical User Interface (GUI).

38. File teks yang diciptakan oleh situs internet untuk menyimpan informasi di komputer user. merupakan informasi seperti preferensi ketika user mengunjungi situs tersebut. merupakan pengertian dari….


File teks yang diciptakan oleh situs internet untuk menyimpan informasi di komputer user, merupakan informasi seperti preferensi ketika user mengunjungi situs tersebut. merupakan pengertian dari:
cookies (atau disebut juga HTTP cookies, atau internet cookies, atau web cookies).
 

Pembahasan

Untuk meningkatkan aspek kebergunaan (usability) dan pengalaman pengguna (user experience) dalam aktivitasnya mengunjungi sebuah situs internet, diperlukan suatu mekanisme di mana situs tersebut dapat menyimpan beberapa informasi kecil pada komputer pengguna. Beberapa informasi kecil ini dapat dibaca oleh situs pada saat pengguna berinteraksi dengan situs tersebut pada saat itu juga atau di waktu mendatang ketika pengguna mengunjungi situs yang sama, yang memungkinkan situs menampilkan interaksi yang sesuai dengan informasi terkait pengguna yang telah disimpan di komputer pengguna.

Informasi-informasi kecil tersebut disimpan dalam media penyimpanan komputer pengguna dalam bentuk file-file teks berukuran kecil, yang dinamakan HTTP cookies, atau bisa disebut juga dengan cookies saja. File-file cookies dapat (namun tidak selalu) memuat informasi otentikasi dan preferensi pengguna pada saat mengunjungi situs.

Beberapa keperluan utama penyimpanan cookies di antaranya adalah:

manajemen sesi (session management), salah satu contohnya adalah penentuan batas waktu maksimum dari status idle ketika user login ke sebuah situs: jika sudah terlalu lama idle, maka pengguna harus login kembali,personalisasi, yang terkait dengan pengalaman user menggunakan situs, dan tracking, untuk menyimpan kecenderungan/kebiasaan pengguna ketika mengunjungi situs.

Karena terkait dengan keamanan data pengguna dan kenyamanan pengguna menjelajah internet, saat ini penggunaan cookies harus melalui persetujuaan pengguna. Ketika pengguna mengunjungi sebuah situs yang belum pernah dikunjungi, dan situs tersebut perlu menyimpan cookies di komputer pengguna, situs akan meminta persetujuan pengguna terlebih dahulu, atau pemberitahuan bahwa situs akan menyimpan cookies (terutama pada situs-situs di mana user harus login terlebih dahulu). Jadi, pengguna harus sadar dan tahu benar bahwa situs yang sedang dikunjungi menyimpan cookies.
